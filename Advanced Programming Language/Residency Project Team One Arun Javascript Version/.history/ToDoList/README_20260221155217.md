## рџљЂ Quick Start

### Prerequisites

- **Node.js** (v14 or higher)
- **npm** (comes with Node.js)
- **Git** (for version control)

### Installation

1. **Clone the Repository**

   ```bash
   git clone <repository-url>
   cd "Master-In-Computer-Science/Advanced Programming Language/Residency Project Team One Arun And Tarak Javascript/TODO"
   ```

2. **Install Dependencies**
   ```bash
   npm install
   ```

### Running the Application

#### Option 1: Run Both Servers Together

```bash
npm run dev
```

This starts both the backend JSON Server (port 3000) and frontend Live Server (port 8080) simultaneously.

> If `npm run dev` fails with an `AbortController is not defined` error in your environment, use **Option 2** (separate terminals).

#### Option 2: Run Servers Separately

**Terminal 1 - Start Backend (JSON Server):**

```bash
npm run server
```

Backend API runs on : `http://localhost:3000`

**Terminal 2 - Start Frontend (Live Server):**

```bash
npm start
```

Frontend runs on: `http://localhost:8080`

`npm start` uses `live-server --ignorePattern=.*db\.json$` to prevent full-page reload/flicker when `db.json` changes. Todo updates still appear through in-app rendering and auto-refresh.

### Access the Application

Open your browser and navigate to:

```
http://localhost:8080
```

**Note:** If port 8080 is busy, Live Server will use an alternative port (e.g., 25298). Check the terminal output for the actual URL.

## рџ”„ How Concurrency Works

### 1. Version Tracking

Every task includes version information:

```json
{
  "id": 1,
  "text": "Learn JavaScript",
  "completed": false,
  "version": 3,
  "lastModifiedBy": 1,
  "lastModifiedAt": "2026-02-20T10:05:30Z",
  "createdAt": "2026-02-20T10:00:00Z"
}
```

### 2. Conflict Detection

When User A and User B edit the same task:

```
T=0:00  Both users read task v1
T=0:01  User A edits в†’ v2 (succeeds)
T=0:02  User B edits в†’ attempts v2 (but server has v2)
        вќЊ Conflict detected!
        User B shown alert
        UI auto-refreshes to show v2
```

### 3. Real-Time Sync

Every 5 seconds:

- Fetches all tasks from server
- Compares with local cache
- Detects ADDED, MODIFIED, DELETED
- Auto-updates UI
- No manual refresh needed

## рџђ› Troubleshooting

| Issue                      | Solution                                                 |
| -------------------------- | -------------------------------------------------------- |
| **Port already in use**    | Change port in package.json or close conflicting process |
| **Tasks not syncing**      | Wait 5 seconds for polling, check console for errors     |

| **db.json not updating**   | Ensure JSON Server is running on port 3000               |
| **UI not refreshing**      | Check browser cache, try Ctrl+Shift+Delete               |
