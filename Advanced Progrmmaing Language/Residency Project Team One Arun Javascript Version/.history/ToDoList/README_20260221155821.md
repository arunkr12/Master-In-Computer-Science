# 📋 Multi-User ToDoList Application with Concurrency Support

A production-ready ToDoList application built with **vanilla JavaScript, Bootstrap 5, Tailwind CSS, and JSON Server** that supports multiple concurrent users with real-time synchronization and conflict detection.

## ✨ Features

### Core Functionality

- ✅ **Multi-User Support** - 3 sample users (Arun, Tarak, Admin) with independent task lists
- ✅ **Task Management** - Create, Read, Update, Delete (CRUD) operations
- ✅ **Task Categorization** - 4 predefined categories (Work, Personal, Education, Project)
- ✅ **Status Tracking** - Mark tasks as completed or pending
- ✅ **Task Prioritization** - High, Medium, Low priority levels
- ✅ **Task Assignment** - Assign tasks to different users
- ✅ **Due Dates** - Set optional deadlines for tasks

### Concurrency Support ⭐

- ✅ **Version Tracking** - Every task tracked with incremental version numbers
- ✅ **Optimistic Locking** - Conflict prevention for simultaneous edits
- ✅ **Conflict Detection** - Real-time detection when multiple users edit same task
- ✅ **Real-Time Sync** - Auto-polling every 60 seconds detects changes from other users
- ✅ **Auto-Recovery** - Automatic refresh and recovery on conflicts
- ✅ **Last-Write-Wins** - Conflict resolution strategy ensures no data loss

### User Interface

- ✅ **Responsive Design** - Bootstrap 5 for mobile-friendly layout
- ✅ **Color-Coded Categories** - Visual distinction for different task categories
- ✅ **User Selector** - Easy switching between different user accounts
- ✅ **Statistics Panel** - Task count summary by status and category
- ✅ **Modern Styling** - Tailwind CSS utilities for polished appearance

## 🚀 Quick Start

### Prerequisites

- **Node.js** (v14 or higher)
- **npm** (comes with Node.js)
- **Git** (for version control)

### Installation

1. **Clone the Repository**

   ```bash
   git clone <repository-url>
   cd "Master-In-Computer-Science/Advanced Programming Language/Residency Project Team One Arun And Tarak Javascript/TODO"
   ```

2. **Install Dependencies**
   ```bash
   npm install
   ```

### Running the Application

#### Option 1: Run Both Servers Together

```bash
npm run dev
```

This starts both the backend JSON Server (port 3000) and frontend Live Server (port 8080) simultaneously.

> If `npm run dev` fails with an `AbortController is not defined` error in your environment, use **Option 2** (separate terminals).

#### Option 2: Run Servers Separately

**Terminal 1 - Start Backend (JSON Server):**

```bash
npm run server
```

Backend API runs on : `http://localhost:3000`

**Terminal 2 - Start Frontend (Live Server):**

```bash
npm start
```

Frontend runs on: `http://localhost:8080`

`npm start` uses `live-server --ignorePattern=.*db\.json$` to prevent full-page reload/flicker when `db.json` changes. Todo updates still appear through in-app rendering and auto-refresh.

### Access the Application

Open your browser and navigate to:

```
http://localhost:8080
```

**Note:** If port 8080 is busy, Live Server will use an alternative port (e.g., 25298). Check the terminal output for the actual URL.

## 🔄 How Concurrency Works

### 1. Version Tracking

Every task includes version information:

```json
{
  "id": 1,
  "text": "Learn JavaScript",
  "completed": false,
  "version": 3,
  "lastModifiedBy": 1,
  "lastModifiedAt": "2026-02-20T10:05:30Z",
  "createdAt": "2026-02-20T10:00:00Z"
}
```

### 2. Conflict Detection

When User A and User B edit the same task:

```
T=0:00  Both users read task v1
T=0:01  User A edits → v2 (succeeds)
T=0:02  User B edits → attempts v2 (but server has v2)
        ❌ Conflict detected!
        User B shown alert
        UI auto-refreshes to show v2
```

### 3. Real-Time Sync

Every 60 seconds:

- Fetches all tasks from server
- Compares with local cache
- Detects ADDED, MODIFIED, DELETED
- Auto-updates UI
- No manual refresh needed

## 🐛 Troubleshooting

| Issue                      | Solution                                                 |
| -------------------------- | -------------------------------------------------------- |
| **Port already in use**    | Change port in package.json or close conflicting process |
| **Tasks not syncing**      | Wait 5 seconds for polling, check console for errors     |
| **Constant conflicts**     | Normal with simultaneous edits - auto-recovery works     |
| **db.json not updating**   | Ensure JSON Server is running on port 3000               |
| **UI not refreshing**      | Check browser cache, try Ctrl+Shift+Delete               |
| **Module not found error** | Run `npm install` to install dependencies                |
