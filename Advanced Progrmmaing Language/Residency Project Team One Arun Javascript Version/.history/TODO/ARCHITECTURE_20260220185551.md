## 🚀 Application Start-Up Flow

### Level 1: Initial Load

```
Browser Load (http://127.0.0.1:8080)
         ↓
    ↙─────────────────────────────────────────────────────────┐
    index.html                                                 │
    ├── Loads Bootstrap CSS (CDN)                             │
    ├── Loads Tailwind CSS (CDN)                              │
    ├── Creates <div id="app"></div>                          │
    └── Loads app.js (module script)                          │
         ↓
```

### Level 2: App Initialization (app.js)

```
app.js Execution
   ↓
1. Import ToDoContainer
2. Get DOM element (#app)
3. Create ToDoContainer instance
4. Call initializeApp() [ASYNC]
   ├── Await todoContainer.initialize()
   │   └── Fetch todos from JSON Server
   ├── Render initial UI
   └── Setup event listeners
   ↓
```

### Level 3: ToDoContainer Initialization

```
ToDoContainer.initialize()
   ↓
1. Call this.todoService.getTodos()
   ├── Make GET request to http://localhost:3000/todos
   ├── Parse JSON response
   └── Return todos array
2. Store todos in this.todos
3. Call updateComponents()
   ├── Pass todos to ToDoList component
   └── Pass todos to Stats component
   ↓
```

### Level 4: Service Layer (TodoService.getTodos)

```
TodoService.getTodos()
   ↓
1. Fetch from "http://localhost:3000/todos"
   ├── GET request
   ├── Wait for response
   └── Parse JSON
2. Return array of todo objects:
   [
     { id: 1, text: "Learn JavaScript", completed: false },
     { id: 2, text: "Build a TODO App", completed: true },
     { id: 3, text: "Master Bootstrap & Tailwind", completed: false }
   ]
   ↓
```

### Level 5: JSON Server Response

```
JSON Server (Port 3000)
   ↓
1. Receives GET /todos
2. Reads db.json file
3. Parses JSON content
4. Returns todos array as JSON response
   ↓
```

## 🔄 User Action Flow - Add Todo Example

```
USER CLICKS "ADD TASK" BUTTON
          ↓
┌─────────────────────────────────────────────────────────────┐
│ LEVEL 1: UI EVENT HANDLER (app.js)                         │
├─────────────────────────────────────────────────────────────┤
│ addBtn.addEventListener('click', async () => {              │
│   1. Get input value from <input class="add-todo-input">    │
│   2. Validate text is not empty                             │
│   3. Call todoContainer.addTodoItem(text)                   │
│ })                                                          │
└─────────────────────────────────────────────────────────────┘
          ↓
┌─────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────┤
│ async addTodoItem(text)                                     │
│ {                                                           │
│   1. Validate text.trim()                                   │
│   2. Call await this.todoService.addTodo(text)             │
│   3. If successful:                                         │
│      - Push to this.todos array                             │
│      - Call updateComponents()                              │
│      - Update UI                                            │
│ }                                                           │
└─────────────────────────────────────────────────────────────┘
          ↓
┌─────────────────────────────────────────────────────────────┐
│ LEVEL 3: SERVICE LAYER (TodoService)                       │
├─────────────────────────────────────────────────────────────┤
│ async addTodo(text)                                         │
│ {                                                           │
│   1. Create request body:                                   │
│      {                                                      │
│        text: "New Task",                                    │
│        completed: false                                     │
│      }                                                      │
│   2. POST to http://localhost:3000/todos                    │
│   3. Set Content-Type: application/json                     │
│   4. Parse response → new todo object with ID               │
│   5. Return { id: 4, text: "...", completed: false }        │
│ }                                                           │
└─────────────────────────────────────────────────────────────┘
          ↓
┌─────────────────────────────────────────────────────────────┐
│ LEVEL 4: JSON SERVER (Port 3000)                           │
├─────────────────────────────────────────────────────────────┤
│ POST /todos                                                 │
│ {                                                           │
│   1. Receive POST request with body                         │
│   2. Generate new ID (auto-increment)                       │
│   3. Add to todos array in memory                           │
│   4. Write updated array to db.json                         │
│   5. Return created object with ID                          │
│ }                                                           │
└─────────────────────────────────────────────────────────────┘
          ↓
┌─────────────────────────────────────────────────────────────┐
│ LEVEL 5: DATA PERSISTENCE (db.json)                        │
├─────────────────────────────────────────────────────────────┤

│   "todos": [                                                │

RESPONSE RETURNS TO LEVEL 3 → LEVEL 2 → LEVEL 1
```

## 🔗 Data Flow for Each Operation

### 1. **FETCH/GET - Load All Todos**

```
User visits app → Browser loads index.html
                    ↓
                  app.js
                    ↓
          initializeApp() [async]
                    ↓
        todoContainer.initialize()
                    ↓
        todoService.getTodos()
                    ↓
        fetch("http://localhost:3000/todos")
                    ↓
        JSON Server: Read db.json
                    ↓
        Return todos array
                    ↓
        Store in this.todos
                    ↓
        updateComponents() → render UI
```

### 2. **CREATE/POST - Add New Todo**

```
User types text + clicks "Add"
                    ↓
        Event listener triggers
                    ↓
        todoContainer.addTodoItem(text)
                    ↓
        todoService.addTodo(text)
                    ↓
        fetch(POST, {text, completed: false})
                    ↓
        JSON Server: Generate ID, add to db.json
                    ↓
        Return new todo object with ID
                    ↓
        Push to this.todos array
                    ↓
        updateComponents() → re-render
```

### 3. **UPDATE/PATCH - Edit or Toggle Todo**

```
User clicks Edit/Checkbox
                    ↓
        Event listener triggers
                    ↓
        todoContainer.editTodoItem(id, text)
        OR
        todoContainer.toggleTodoItem(id)
                    ↓
        todoService.updateTodo(id, {updates})
                    ↓
        fetch(PATCH, {text or completed})
                    ↓
        JSON Server: Update db.json entry
                    ↓
        Return updated todo object
                    ↓
        Update local this.todos array
                    ↓
        updateComponents() → re-render
```

### 4. **DELETE/DELETE - Remove Todo**

```
User clicks Delete button
                    ↓
        Event listener triggers
                    ↓
        todoContainer.deleteTodoItem(id)
                    ↓
        todoService.deleteTodo(id)
                    ↓
        fetch(DELETE, /todos/{id})
                    ↓
        JSON Server: Remove from db.json
                    ↓
        Return success
                    ↓
        Filter out from this.todos array
                    ↓
        updateComponents() → re-render
```

## 📝 API Endpoints

```
Base URL: http://localhost:3000

GET    /todos              - Fetch all todos
POST   /todos              - Create new todo
PATCH  /todos/{id}         - Update specific todo
DELETE /todos/{id}         - Delete specific todo

Examples:
GET    /todos/1            - Get todo with id 1
PATCH  /todos/1            - Update todo 1 (body: {completed: true})
```

## 🏗️ Architecture Patterns Used

1. **MVC Pattern** - Separation of concerns
   - Model: TodoService (data)
   - View: Components (Header, ToDoList, etc.)
   - Controller: ToDoContainer (business logic)

2. **Service Layer Pattern** - TodoService abstracts API calls

3. **Component-Based Architecture** - Modular, reusable components

4. **Async/Await** - Clean asynchronous operation handling

5. **REST API** - Standard HTTP methods for CRUD operations
